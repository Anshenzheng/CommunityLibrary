# 社区图书角借阅管理系统 - 启动与测试说明

## 项目概述

本项目是一个社区图书角借阅管理系统，采用前后端分离架构：
- **后端**：Java + Spring Boot 3 + MySQL
- **前端**：Angular 17 + Bootstrap 5

界面设计清新简约，具有书香氛围，支持读者和管理员两种角色。

---

## 环境要求

### 后端环境
- JDK 17 或更高版本
- Maven 3.6+
- MySQL 8.0 或更高版本

### 前端环境
- Node.js 18.x 或更高版本
- npm 9.x 或更高版本 (随 Node.js 安装)
- Angular CLI 17.x (可选，用于开发)

## 安装步骤

### 1. 数据库配置

#### 1.1 创建数据库
```sql
CREATE DATABASE community_library CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

#### 1.2 导入初始数据
执行数据库初始化脚本：
```
backend/src/main/resources/db/init.sql
```

该脚本包含：
- 数据表结构创建
- 初始分类数据（8个图书分类）
- 测试管理员账户（用户名: `admin`, 密码: `admin123`）
- 测试读者账户（用户名: `reader1`/`reader2`, 密码: `user123`）
- 8本测试图书数据

#### 1.3 修改数据库连接配置
编辑 `backend/src/main/resources/application.properties`：
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/community_library?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai&allowPublicKeyRetrieval=true
spring.datasource.username=root
spring.datasource.password=root  # 修改为您的MySQL密码
```

### 2. 后端依赖安装

进入后端目录：
```bash
cd backend
```

使用 Maven 安装依赖：
```bash
mvn clean install
```

或者使用 Maven Wrapper（如果已配置）：
```bash
./mvnw clean install  # Linux/macOS
mvnw.cmd clean install  # Windows
```

### 3. 前端依赖安装

进入前端目录：
```bash
cd frontend
```

使用 npm 安装依赖：
```bash
npm install
```

## 启动步骤

### 方式一：开发模式（推荐用于测试）

#### 1. 启动后端服务

进入后端目录：
```bash
cd backend
```

启动 Spring Boot 应用：
```bash
mvn spring-boot:run
```

或者运行编译后的 jar 包：
```bash
mvn package
java -jar target/library-1.0.0.jar
```

后端服务默认运行在 **http://localhost:8080**

#### 2. 启动前端服务

打开新的终端窗口，进入前端目录：
```bash
cd frontend
```

启动 Angular 开发服务器：
```bash
npm start
```

前端服务默认运行在 **http://localhost:4200**

### 方式二：生产模式

#### 1. 构建后端
```bash
cd backend
mvn clean package
```

生成的 jar 包位于 `backend/target/library-1.0.0.jar`

#### 2. 构建前端
```bash
cd frontend
npm run build
```

生成的静态文件位于 `frontend/dist/community-library/`

#### 3. 部署
- 将后端 jar 包部署到 Java 应用服务器
- 将前端静态文件部署到 Nginx 或其他 Web 服务器
- 配置反向代理，将 `/api` 请求转发到后端服务

## 系统功能测试

### 测试账户

| 角色 | 用户名 | 密码 | 说明 |
|------|--------|------|------|
| 管理员 | admin | admin123 | 拥有全部管理权限 |
| 读者 | reader1 | user123 | 普通读者账户 |
| 读者 | reader2 | user123 | 普通读者账户 |

### 测试流程

#### 1. 基础功能测试

1. **访问系统**
   - 打开浏览器，访问 http://localhost:4200
   - 应该看到首页，显示推荐图书和分类

2. **浏览图书**
   - 点击导航栏「图书浏览」
   - 测试搜索功能（按书名、作者、ISBN）
   - 测试分类筛选
   - 点击图书查看详情页

3. **读者注册**
   - 点击「注册」按钮
   - 填写注册信息（用户名、密码、真实姓名等）
   - 提交后跳转到登录页

4. **读者登录**
   - 使用 reader1/user123 登录
   - 登录成功后跳转首页
   - 导航栏显示用户信息和「我的借阅」「个人中心」

5. **借阅申请**
   - 浏览图书，选择一本状态为「可借」的图书
   - 点击「申请借阅」按钮
   - 系统提示「借阅申请已提交，请等待管理员审核」

6. **查看借阅记录**
   - 点击「我的借阅」
   - 可以看到刚才的借阅申请，状态为「待审核」
   - 测试状态筛选功能

#### 2. 管理员功能测试

1. **管理员登录**
   - 退出当前登录（点击右上角用户头像 → 退出登录）
   - 使用 admin/admin123 登录
   - 导航栏显示「管理后台」菜单

2. **数据概览**
   - 点击「管理后台」→「数据概览」
   - 查看统计数据（用户数、图书数、借阅数等）
   - 测试快捷操作按钮

3. **借阅审核**
   - 点击「管理后台」→「借阅审核」
   - 可以看到刚才读者提交的借阅申请
   - 点击「批准」或「拒绝」
     - 批准：借阅状态变为「借阅中」，图书库存减1
     - 拒绝：需要填写拒绝原因
   - 测试搜索和筛选功能
   - 测试导出 Excel 功能

4. **图书管理**
   - 点击「管理后台」→「图书管理」
   - 测试添加新图书（点击「添加图书」按钮）
   - 测试编辑图书（点击「编辑」按钮）
   - 测试删除图书（点击「删除」按钮，确认弹窗）
   - 测试搜索和分类筛选
   - 测试导出 Excel 功能

5. **用户管理**
   - 点击「管理后台」→「用户管理」
   - 查看用户列表
   - 测试修改用户状态（启用/禁用）
   - 测试修改用户角色（读者/管理员）
   - 测试搜索和筛选
   - 测试导出 Excel 功能

6. **统计分析**
   - 点击「管理后台」→「统计分析」
   - 查看统计数据
   - 测试日期范围筛选
   - 测试导出 Excel 功能

#### 3. 还书与逾期测试

1. **还书功能**
   - 使用读者账户登录
   - 点击「我的借阅」
   - 找到状态为「借阅中」的记录
   - 点击「还书」按钮
   - 确认后，状态变为「已归还」，图书库存加1

2. **逾期自动标记**
   - 系统每天凌晨2点会自动检查逾期记录
   - 当借阅记录超过应还日期时，状态自动变为「已逾期」
   - 还书时会计算逾期罚款（每天0.5元）

curl http://localhost:8080/api/users/me \

## 常见问题

### 1. 数据库连接失败
- 检查 MySQL 服务是否启动
- 检查用户名、密码是否正确
- 检查数据库是否已创建
- 检查端口是否正确（默认3306）

### 2. 前端无法连接后端
- 检查后端服务是否启动（默认8080端口）
- 检查前端 `environment.ts` 中的 `apiUrl` 配置
- 检查浏览器控制台是否有 CORS 错误

### 3. 登录失败
- 检查数据库是否已导入初始数据
- 检查密码加密方式（后端使用 BCrypt）
- 检查用户状态是否为「正常」

### 4. 图书库存显示异常
- 检查借阅记录状态
- 检查图书库存更新逻辑
- 查看后端日志是否有错误
